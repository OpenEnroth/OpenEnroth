Detailed Description

This component can be used to make the engine more deterministic. It replaces the system timer with a fake one that advances by 16ms on each frame, and also resets the global random number generator.

Note that this component is intentionally very dumb. All the methods just do exactly what you would expect right away, so where you're calling them from becomes very important. E.g. calling startDeterministicSegment from an event handler would probably make little sense.

◆ startDeterministicSegment()

void EngineDeterministicComponent::startDeterministicSegment ( int  frameTimeMs)

Starts a new deterministic segment, resetting all state - rolls the tick count back to zero, and resets grng.

Note on why we're passing frame time in milliseconds here and not setting the target fps directly.

It is possible to target e.g. 60fps, but since the platform API is in milliseconds, this will result in staggered frame times, with every 1st and 2nd frame taking 17ms, and every 3rd one taking 16ms. This might result in some non-determinism down the line, e.g. changing the code in level loading will change the number of frames it takes to load a level, and this will shift the timing sequence for the actual game frames after the level is loaded. Unlikely to really affect anything, but we'd rather not find out.

Parameters
frameTimeMsFrame time to use inside the segment, in milliseconds.
