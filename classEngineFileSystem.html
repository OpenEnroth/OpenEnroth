Detailed Description

File system manager as used by the engine.

Default data fs configuration:

- MergingFS
- LowercaseFS
- DirectoryFS for <data-path>.
- EmbeddedFS for scripts and shaders.

User fs is just a DirectoryFS.

Effectively user fs is the write interface, while data fs is the read interface for everything. Writing through the data fs will fail.

It's possible to replace both user and data filesystems, dataFs and userFs are proxies that you can call setBase on. To reset the proxies to their default state, call dataFs()->setBase(defaultDataFs()), and the same for the user fs.
